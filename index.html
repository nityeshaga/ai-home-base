Concepts explained

For the curious. Click to expand.

What is it? SSH (Secure Shell) lets you open a terminal on another computer over the internet. You type commands on your laptop, they execute on the Mac Mini. Text only — no desktop, no mouse. Think of it as a phone call to the Mac Mini's command line.

Why do we need it? To install software, check logs, restart services, and manage everything on the Mac Mini without being physically there. It's built into macOS — just flip a switch in System Settings.
What is it? Tailscale creates an invisible, encrypted tunnel between your devices. Your MacBook in India and the Mac Mini in New York join the same private network — as if they were on the same WiFi. Each device gets a stable address (like 100.x.x.x) that never changes.

Why do we need it? The Mac Mini sits behind an office router with no public address. Without Tailscale, you'd need to configure port forwarding, get a static IP, and punch holes in the firewall — complicated and insecure. Tailscale skips all of that. Install the app, sign in, done. Free for personal use.

Analogy: If SSH is a phone call, Tailscale is the phone network. SSH needs a way to reach the Mac Mini — Tailscale provides that path.
What is it? When you SSH into the Mac Mini, your session dies if the connection drops (laptop closes, WiFi blips). tmux keeps your session alive on the Mac Mini regardless. Disconnect, reconnect hours later — everything is exactly where you left it.

Why do we need it? For interactive Claude Code sessions. If you're SSH'd in working with Claude Code directly and your internet hiccups, tmux means you don't lose your work. For the Slack bots, tmux isn't needed — launchd handles that.
What is it? macOS's built-in system for managing background services. You write a small config file that says "run this script, keep it alive, restart it if it crashes, start it when the computer boots." Apple uses it for its own services.

Why do we need it? This is what makes the Slack bots truly always-on. Bot crashes at 3am? launchd restarts it in seconds. Mac Mini reboots after a power outage? launchd starts all the bots automatically. No human intervention needed.
What is it? Normally, Slack bots need a public URL that Slack sends messages to. Socket Mode flips this — our bot reaches out to Slack and holds the connection open. Messages flow through that connection. No public URL, no firewall config, no exposed ports.

Why do we need it? The Mac Mini is behind a private network (Tailscale). It has no public URL. Socket Mode means the bot works perfectly behind any network — it only makes outbound connections, which always work.
What is it? A drop-in replacement for SSH designed for mobile and high-latency connections. It shows your keystrokes instantly (doesn't wait for the server), survives WiFi switches, and handles packet loss gracefully. In tests, it reduced response time from 16.8 seconds to 0.33 seconds on a lossy link.

Why do we need it? India to New York is ~250ms latency. With regular SSH, every keystroke takes a quarter-second round trip — feels sluggish. Mosh makes it feel local. Install on both machines, use mosh instead of ssh.
What is it? The claude command-line tool from Anthropic. When the Slack bot receives a message, it runs claude -p "your message" — which gives Claude access to all your files, MCP servers (Google Workspace, Granola, etc.), plugins, and CLAUDE.md context. Powered by the $200/month Max subscription — flat rate, no per-message API costs.

Why this and not the API? The CLI inherits everything — your OAuth sessions, plugin marketplace, installed MCP servers, project context. The API would require rebuilding all of that from scratch.
Phase 1 Natalia

Physical setup

One-time, in-person. ~20 minutes.

Hey Natalia!

This is your section. You're setting up the Mac Mini so Nityesh can take over remotely from India. Once you're done here, you won't need to touch it again unless there's a hardware issue. Everything below is a checklist — work through it top to bottom.

1. Plug it in

  • Connect the Mac Mini to power
  • Connect to the office ethernet (the cable that plugs into the wall/router — WiFi works but ethernet is more reliable for a server)
  • Connect a monitor, keyboard, and mouse temporarily (you'll disconnect these when done)
  • Optional: plug in an HDMI dummy plug (~$10 on Amazon) — this tricks the Mac into thinking a monitor is connected, which prevents some graphics quirks when running headless

2. Initial macOS setup

  • Power on and go through the macOS setup wizard
  • Create a user account — use a name like every or nityesh (this will be the account used for everything)
  • Sign in with an Apple ID (needed for App Store / Tailscale installation)
  • When asked about FileVault (disk encryption): skip it / turn it off
    Why? If the Mac Mini restarts after a power outage, FileVault shows an unlock screen requiring a physical keyboard. Since nobody will be there to type the password, the machine would be stuck. The trade-off is worth it — the Mac is in a secure office and behind Tailscale.

3. System Settings checklist

Open System Settings and configure each of these:

Energy
  • Prevent automatic sleeping when the display is off — toggle ON
  • Start up automatically after a power failure — toggle ON
  • Wake for network access — toggle ON
General → Sharing
  • Remote Login (SSH) — toggle ON. Note the username shown (e.g., every@macmini.local)
  • Screen Sharing — toggle ON (so Nityesh can see the full desktop remotely when needed)
Users & Groups
  • Automatic login — set to the user account you created (so the Mac logs in by itself after a restart)

4. Install Tailscale

Tailscale is the app that connects the Mac Mini to Nityesh's laptop securely.

  • Open the Mac App Store, search for Tailscale, install it
  • Open Tailscale, sign in with the shared account (Nityesh will give you the login — Google, GitHub, or Apple ID)
  • Once connected, Tailscale shows an IP address like 100.x.x.x — send this to Nityesh
  • In Tailscale preferences, enable "Start Tailscale when I log in"

5. Install Homebrew

Homebrew is a package manager — it lets us install tools easily from the terminal. Nityesh needs this to install everything else remotely.

  • Open Terminal (search for it in Spotlight — Cmd+Space, type "Terminal")
  • Paste this command and press Enter:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your password (the one you set during macOS setup). This takes a few minutes.

6. Wrap up

  • Send Nityesh: the Tailscale IP, the username, and the password for the account you created
  • Disconnect the monitor, keyboard, and mouse — the Mac Mini runs headless from here
  • Leave it powered on and connected to ethernet
You're done! Nityesh takes over from here. The Mac Mini will sit quietly in the office, always on, doing its thing. You'll be able to DM the AI teammates in Slack once everything is set up.

Optional (nice to have)

  • Smart plug (~$15, TP-Link Kasa) — lets Nityesh power-cycle the Mac Mini remotely from India if it ever freezes
  • Small UPS (~$50) — battery backup that keeps the Mac Mini alive during brief power outages

Phase 3 Nityesh

Make it always on

The bot should survive crashes, reboots, and disconnections — forever.

1. Create a launchd service

This tells macOS to keep the Slack bot running at all times.

# Create the plist file
cat <<'EOF' > ~/Library/LaunchAgents/com.every.claudie-slack.plist
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
"http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
<key>Label</key>
<string>com.every.claudie-slack</string>
<key>ProgramArguments</key>
<array>
<string>/Users/every/Documents/Github/every-consulting/slack-bot/venv/bin/python</string>
<string>/Users/every/Documents/Github/every-consulting/slack-bot/claudie_bot.py</string>
</array>
<key>WorkingDirectory</key>
<string>/Users/every/Documents/Github/every-consulting/slack-bot</string>
<key>RunAtLoad</key><true/>
<key>KeepAlive</key><true/>
<key>StandardOutPath</key>
<string>/Users/every/Documents/Github/every-consulting/slack-bot/claudie.log</string>
<key>StandardErrorPath</key>
<string>/Users/every/Documents/Github/every-consulting/slack-bot/claudie-stderr.log</string>
</dict>
</plist>
EOF

2. Load and verify

# Load the service
launchctl load ~/Library/LaunchAgents/com.every.claudie-slack.plist

# Verify it's running
launchctl list | grep claudie

# Watch the logs
tail -f ~/Documents/Github/every-consulting/slack-bot/claudie.log

3. Test crash recovery

# Find the process ID
pgrep -f claudie_bot

# Kill it — launchd should restart it within seconds
kill $(pgrep -f claudie_bot)

# Check it came back
sleep 3 && pgrep -f claudie_bot

If the process ID changed, launchd restarted it. DM Claudie in Slack to confirm it responds.

4. Test reboot survival

# Reboot the Mac Mini remotely
sudo reboot

# Wait 2-3 minutes, then SSH back in
ssh every@100.x.x.x

# Check the bot is running
pgrep -f claudie_bot

DM Claudie in Slack. If it responds — you have an always-on AI assistant.

It's alive! The Mac Mini now runs your AI team 24/7. It survives crashes, reboots, and power outages. You can manage everything from India.

What's next

Once the foundation is solid, the possibilities open up.

One bot per AI employee

Today there's one Slack bot (Claudie). The goal: each plugin becomes its own Slack app — DM Jean-Claude for sales, Claudie for PM, just like DMing a real teammate. Same bot code, different config per identity.

Browser automation

Install Playwright on the Mac Mini. The AI can control a real browser — log into apps, fill forms, take screenshots. Anything you can do in Chrome, it can do. Great for tasks that require authentication or interacting with web UIs that don't have APIs.

Scheduled jobs

Morning briefings, pipeline refreshes, automated reports — all running on a schedule via launchd cron jobs. The bot's --send flag already supports proactive DMs. Wire it to a schedule and you get a daily briefing in Slack every morning.

Full desktop access

Screen Sharing over Tailscale gives you full GUI access to the Mac Mini — mouse, windows, everything. Open it via vnc://100.x.x.x from Finder. Useful for logging into apps, managing OAuth sessions, or anything that needs a browser window.

Multi-user access

Add more Slack user IDs to AUTHORIZED_USERS in the .env file. Anyone on the team can talk to the AI co-workers. The audit log tracks who said what.

Good to know

Practical limitations and how to handle them.

Scenario What happens
Power outage Mac Mini restarts automatically. launchd relaunches the bots. Downtime: 2-3 minutes. A UPS ($50) covers brief outages entirely.
Mac Mini freezes Smart plug lets you power-cycle remotely from India. If no smart plug, Natalia presses the power button.
Internet outage (NYC) Nothing you can do remotely. The Slack bot reconnects automatically when internet returns. A USB LTE dongle (~$30/mo) provides cellular backup if this is a concern.
Claude Max rate limits ~900 messages per 5-hour rolling window. Plenty for normal use. If you hit limits, Max lets you buy overflow at API rates — no hard wall.
SSH latency (India→NYC) ~250ms. Noticeable when typing in terminal. Use Mosh for a smoother experience. For Slack, latency is irrelevant — you type in Slack, wait for a response.
macOS updates Minor updates: trigger via SSH (sudo softwareupdate -i -a). Major upgrades may need someone physically present.
